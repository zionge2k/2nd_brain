# KISS & DRY 코드룰 (핵심 요약)

## 정의
- **KISS**: 코드의 의도가 명확하게 드러나도록 단순하게 작성
- **DRY**: 3번 이상 반복되는 로직은 공통화하여 중복 제거

## 핵심 판단 기준

### 1. 인지적 복잡도 제한
- 중첩 깊이 3단계 이하 유지
- 한 기능 이해에 5개 이상 파일 참조 금지
- 맥락을 계속 기억해야 하는 중첩 구조 피하기

### 2. DRY 적용 시점 (3의 규칙)
- 2번까지 중복 허용
- 3번째 반복 시 추상화 고려
- 단, 함께 변경되는 로직인지 확인 필수

### 3. 과도한 추상화 방지
- 현재 존재하는 중복에만 DRY 적용
- 미래 가능성만으로 추상화 금지 (YAGNI)
- 추상화로 인지적 복잡도 증가 시 중복 허용

## 체크포인트
```
✓ 주석 없이 의도 파악 가능한가?
✓ 함수는 단일 책임을 가지는가?
✓ 같은 로직이 3번 이상 반복되는가?
✓ 중복 제거 후 오히려 이해가 어려워지지 않는가?
```

## 우선순위
**의도의 명확성 > 중복 제거 > 코드 줄 수**
